## 2025-06-30 - Git Argument Injection and HTTP Resource Safety
**Vulnerability:** User-controlled Git references (branches/tags/SHAs) and paths were passed to `exec.Command` without the `--` separator, allowing for potential argument injection if a reference started with a hyphen. Additionally, `http.DefaultClient` was used for remote API calls, which lacks timeouts and could lead to resource exhaustion (DoS).
**Learning:** Even when inputs are expected to be well-formed Git references, defensive programming using the `--` separator is necessary to prevent them from being interpreted as CLI flags. Resource management (timeouts and context propagation) is equally critical for security in networked applications to prevent denial-of-service through resource exhaustion.
**Prevention:** Always use the `--` separator in `git` commands to delineate options from positional arguments. Replace `http.DefaultClient` with a custom client that enforces reasonable timeouts (e.g., `constants.DefaultHTTPClientTimeout`) and prefer `exec.CommandContext` to ensure child processes respect request lifecycles.
