## 2025-07-02 - [Optimizing Fuzzy Matching with Single-Row DP and Stack Allocation]
**Learning:** For fuzzy matching (Levenshtein distance) on relatively short strings like command names or workflow identifiers, the memory allocation overhead can exceed the actual computation cost. Swapping inputs to ensure the shorter string determines the DP row size, combined with a stack-allocated buffer (e.g., `[65]int`), eliminates heap allocations entirely for most common cases. Additionally, an early-exit check in the outer fuzzy match loop based on string length differences (since `distance >= abs(len(a) - len(b))`) can skip the expensive DP calculation for ~50% of candidates.
**Action:** Always prefer stack-allocated buffers for small, short-lived slices in high-frequency utility functions. Implement early-exit checks based on simple metrics (like string length) before entering O(N*M) loops.
